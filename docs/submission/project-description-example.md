# Project Description Example

This example is tailored to the current `tokenizers` app so it can be adapted
directly for submission.

## Project Name

`Tokenizers`

## Brief Description

Tokenizers is a local-first medication tracker that helps people turn messy
medication information into a clear, usable schedule. Users can add or update
medications manually or use the assistant to propose changes from natural
language or prescription photos, with explicit review required before anything
is applied. The app keeps data on-device, supports reminders, and gives users a
safer way to manage medication changes without relying on a chat-only workflow.

## Project Outcome

The outcome of this project is a more private and practical way for people to
manage medication routines. By combining assistant-generated drafts with a
clear human review step, the app reduces confusion around schedules while
keeping the user in control of what gets saved. This can improve day-to-day
adherence, lower the risk of misunderstanding medication instructions, and make
health tracking more accessible for people who struggle with fragmented or
hard-to-read medication information.
